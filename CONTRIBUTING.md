# 贡献指南

感谢您对本仓库的关注和贡献意愿！我们欢迎所有形式的贡献，包括但不限于：

- ✍️ 新增疾病知识内容
- 🔍 完善现有内容
- 🐛 修正错误或过时信息
- 📝 改进文档结构和排版
- 🌐 翻译内容到其他语言
- 💡 提供建议和意见

---

## 如何贡献

### 1. 新增疾病内容

#### 步骤 1：使用内容模板
请使用我们的标准内容模板（见下文），确保内容的格式和结构统一。

#### 步骤 2：创建新文件
在相应的分类目录下创建新的 Markdown 文件。

示例：
```bash
# 在心血管分类下创建高血压文档
cardiovascular/hypertension.md
```

#### 步骤 3：填写内容
按照模板填写疾病相关信息，确保内容准确、完整。

#### 步骤 4：添加到目录索引
更新相应的 README.md，添加新文档的链接。

### 2. 修改现有内容

#### 步骤 1：Fork 仓库
如果您还没有 Fork，请先 Fork 本仓库。

#### 步骤 2：创建分支
```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/issue-number
```

#### 步骤 3：修改内容
在您的新分支上进行修改。

#### 步骤 4：提交修改
```bash
git add .
git commit -m "简短描述您的修改"
```

#### 步骤 5：提交 Pull Request
将您的修改推送回您的 Fork，然后提交 Pull Request 到主仓库。

## 内容质量标准

### ✅ 我们期望的内容

1. **准确性**
   - 信息来源可靠，优先引用权威医学机构
   - 避免错误或误导性的信息
   - 标注信息来源和更新时间

2. **完整性**
   - 涵盖疾病的各个方面（病因、症状、诊断、治疗、预防）
   - 内容结构清晰，易于理解

3. **可读性**
   - 使用简洁明了的语言
   - 避免过度专业术语，或对术语进行解释
   - 使用适当的格式（标题、列表、表格等）提升可读性

4. **时效性**
   - 确保信息是最新的
   - 定期更新过时内容
   - 标注最后更新日期

### ❌ 我们不接受的内容

1. **明显错误的信息**
   - 与权威医学指南相悖的内容
   - 缺乏证据支持的夸大宣传

2. **商业推广**
   - 某些产品或服务的广告
   - 带有商业目的的软文

3. **版权内容**
   - 直接复制受版权保护的内容
   - 未经授权的转载

4. **误导性信息**
   - 可能导致读者延误就医的内容
   - 过度承诺疗效的内容

## 分支命名规范

使用清晰的分支名称，便于追踪：

- 新增内容：`feature/disease-name`
- 修复问题：`fix/issue-number` 或 `fix/description`
- 更新内容：`update/disease-name`
- 文档改进：`docs/improvement-description`

## Commit 信息规范

使用清晰、简洁的 Commit 信息：

```
新增：添加高血压疾病知识
修复：修正便秘文档中的药物用法错误
更新：更新脑梗文档的最新治疗指南
文档：统一格式和排版
```

## PR（Pull Request）规范

### PR 标题
简洁明了，描述清楚本次修改的内容。

### PR 描述
请包括：

1. **修改内容**：简述您做了哪些修改
2. **修改原因**：为什么做这些修改
3. **相关 Issue**：如果有相关 Issue，请关联
4. **测试情况**：如果是代码修改，请说明测试情况

### PR 检查清单

在提交 PR 前，请确认：

- [ ] 已阅读并遵守贡献指南
- [ ] 已使用标准内容模板
- [ ] 内容信息准确，来源可靠
- [ ] 已更新相关索引和链接
- [ ] 已更新最后更新时间
- [ ] 已进行拼写和语法检查
- [ ] Markdown 格式正确

## 代码风格

虽然本仓库主要是 Markdown 文档，但如果有代码部分，请遵循：

- 使用 2 个空格缩进
- 每行不超过 100 个字符
- 使用有意义的变量和函数名
